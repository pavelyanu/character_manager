The main package containing all the classes.

The package contans two runnable classes: {@link org.mff.Server} and {@link org.mff.PlayerClient} as well as class {@link org.mff.Main} which takes either "client" or "server" to run them respectively. Package also contains auxiliary classes.

Server and Database

{@link org.mff.Server} is a class responsible for allocating new thread for the clients as well as governing communication between clients and the server, which in turn communicates with database.

{@link org.mff.Database} is a class responsible for managing compain data and compain metadata. Compain data contains player character, while compain meta data contains player - character relations.

Message and MessageType

{@link org.mff.Message} is a class representing a unit of communication between a server and a client. Message contains the String payload and MessageType message type.

{@link org.mff.MessageType} is an enum representing all possible types of messages.

PlayerClient and Character

{@link org.mff PlayerClient} is a class responsible for client side server client communication as well as client user interface. The command line user interface uses following commands: auth - to reauthenticate, pick - to pick a character, view - to view character, set - to set a field on a character sheet, help - to see the help message.

{@link org.mff Character} is a class representing a player character. It has all the fields which a character in a DnD game would have as well as ability to set certain field. When one field is set all the field depending on it are updated.

External libraries

The package uses Jackson library to convert Character class to JSON and back whenever such conversion is needed.
